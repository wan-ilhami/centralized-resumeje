# ResumeJe AI Agent Instructions

**Custom behavior for all prompts in this workspace**

## Auto-Load on Every Prompt

At the START of EVERY prompt session, AUTOMATICALLY:

1. 📖 Read [CODEBASE_REFERENCE.md](../CODEBASE_REFERENCE.md) (main reference)
2. ✅ Check current code structure (this is a simplified codebase)
3. ✅ Run `bun verify` after ANY code changes

## Core Directives

### 1. Quality First

- ✅ Zero TypeScript errors (strict mode)
- ✅ Zero ESLint warnings
- ✅ All code passes `bun verify`
- ✅ Never use `@ts-ignore`, `any`, or `eslint-disable` without justification

### 2. Workflow

- **Start**: `cd centralized-resume && bun dev` (Backend + Frontend, no dependencies)
- **Verification**: `bun verify` after every change
- **Testing**: Manual test on dev before claiming completion
- **Commits**: Use conventional commit format
- **Documentation**: Update CODEBASE_REFERENCE.md on structural changes

### 3. Tech Stack (Simplified)

- **Frontend**: Next.js 16, React 19, TypeScript, Tailwind CSS v4, Redux Toolkit, Vercel Analytics
- **Backend**: Express 5, TypeScript, Multer
- **AI**: HuggingFace Router (primary) → Groq SDK (fallback)
- **Database**: PostgreSQL 16, Prisma ORM (feedback/messages only)
- **Ports**: Frontend 3000, Backend 3001

## Development Scripts (bun Commands)

### Quick Start (All You Need)

```bash
cd centralized-resume
bun dev
# Starts: Backend (3001) + Frontend (3000)
# Prerequisites: None
# All sync endpoints ready immediately
# Use for: all development
```

**What it starts**:

- ✅ Backend API server (3001)
- ✅ Frontend dev server (3000)

### Verification

```bash
bun verify              # ESLint + TypeScript (both)
bun verify:frontend     # Frontend only
bun verify:backend      # Backend only
bun lint:fix            # Auto-fix linting errors
```

### Build

```bash
bun build:all           # Build both frontend + backend
```

## Environment Setup

### Create `backend/.env`

```env
NODE_ENV=development
PORT=3001
GROQ_API_KEY=gsk_your_key
HUGGINGFACE_API_KEY=hf_your_key
JWT_SECRET=dev_secret
CORS_ORIGIN=http://localhost:3000
```

### Create `frontend/.env`

```env
NEXT_PUBLIC_API_BASE_URL=http://localhost:3001
NEXT_PUBLIC_APP_ENV=development
NEXT_PUBLIC_GOOGLE_CLIENT_ID=your_google_client_id
```
